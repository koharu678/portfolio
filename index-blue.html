1. 产品概述

1.1 产品愿景

让业务部门自己上传数据、自己筛选分析，财务不再需要重复出表。

1.2 一句话描述

一款财务自助分析看板，业务部门拖拽上传 Excel/CSV 即可自动生成 KPI 摘要、趋势图、分类对比图和假设分析（敏感性分析），把财务从每周 2 小时的重复出表中解放出来。

1.3 商业目标

Villy 正在应聘财务 BP 岗位，JD 要求：数据处理能力、财务分析能力、数据决策思维、原型设计能力。本产品作为作品集的第二件，与项目一（SAP 订单健康度监控）构成完整的能力展示闭环：

项目核心能力JD 覆盖
项目一：SAP 订单监控异常监控（发现问题）SAP、SQL、Power BI
项目二：财务自助分析自助分析（辅助决策）财务分析、数据决策、原型设计

1.4 核心叙事（面试故事线）

"原来财务每周花 2 小时给业务部门做重复报表 → 现在业务部门拖拽上传 Excel，工具自动识别数据结构并生成分析图表，加上假设分析滑块，业务部门能自己回答'如果价格涨 10%，利润会变多少'这类问题。"

2. 目标用户

2.1 主要用户：面试官

  • 扫码打开网页，30 秒内理解产品价值
  • 拖入自己的 Excel 文件 → 自动出分析 → 留下深刻印象
  • 拖动假设分析滑块 → 直观感受"数据决策思维"

2.2 次要用户：业务部门员工（设计目标）

  • 不懂 SQL、不懂 BI 工具
  • 有 Excel 数据，想看分析结果
  • 不想每次找财务出表

2.3 辅助用户：财务 BP（Villy 面试角色的日常）

  • 减少重复性出表工作
  • 提供自助服务工具给业务部门
  • 通过假设分析支持管理层决策

3.6 假设分析模块 ⭐ 核心亮点

用户故事：作为业务管理者，我想知道"如果我们把价格提高 10%，利润率会变成多少？"——不需要财务帮我算，我拖动鼠标就知道答案。

需求 ID描述优先级
WIF-01价格调整范围滑块：-20% ~ +20%，步长 1%P0
WIF-02实时显示当前毛利率 vs 调整后毛利率P0
WIF-03实时显示当前利润 vs 调整后利润P0
WIF-04实时显示利润变动百分比（绿涨/红跌/灰色不变）P0
WIF-05KPI 卡片和图表随滑块联动更新P0
WIF-06滑块归零后恢复原始数据P0
WIF-07无收入列时显示"需要收入数据才能进行假设分析"P2

计算逻辑：

调整后收入 = 原始收入 × (1 + 调整比例)
调整后利润 = 调整后收入 - 成本 - 费用
调整后毛利率 = (调整后收入 - 成本) / 调整后收入

5.2 数据流

用户上传文件
→ PapaParse / SheetJS 解析
→ 智能列识别引擎（扫描表头 → 匹配关键词 → 标注列角色）
→ 存入 AppState.rawData
→ applyFilters() 应用筛选 → computeKPIs() 计算 KPI
→ 同步刷新所有视图

用户拖动假设分析滑块
→ AppState.whatIf.priceAdjustment 更新
→ computeKPIs() 重算
→ 同步刷新所有视图

5.3 数据模型

AppState = {
  rawData: [],           // 解析后的原始数据

    expenseCol: null,    // 费用列名

}
